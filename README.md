## Overview
Nom nom is a website that assists in the sharing of recipes, meetups and acts as the intermediary for the meetups themselves. The website is connected to a local database that has a total of 12 tables. Each table has their own purpose, but they all share the same amount of importance in the functionality of the website.

Upon looking up the website up on the web, the user will be shown the landing page.  This page will only show a minimal amount of data, until the user either logs in or registers a new account. All data submitted by the user will be stored in the database in a safe manner. For example, the passwords are stored into the database as hashed to ensure safety of personal data.  

From there, the user is directed to the home page, where they are shown a limited amount of data, but are able to expand upon this information. They are able to click on recipe tab, meetup tab, profile tab, or to logout. They are also given the option to search for a recipe by keyword or they can search for a meetup by keyword.

## Lessons Learned
As we all know, while working in a team environment, communication is of the upmost importance. To avoid any sort of merge conflicts, we were constantly pushing our files to Github, allowing the other members in the group to pull the data and see what has already been done. We learned the value of constantly pushing data very early on in the project and decided it would be in good practice to continue doing this until the project was finished.
